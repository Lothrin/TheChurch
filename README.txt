Project Overview (For German scroll down)
--------------------------------------------------------------------------------------------
Dynamic Event Management System
-Admins can create, edit, delete, and display events.
-Events are split into upcoming and past with real-time updates.
-Users can view detailed event pages with images, descriptions, and booking options.

Admin Dashboard
-Full control over users, reservations, menus, and events.
-CRUD functionality enables easy management of data.
-Supports multiple image uploads for events, enhancing user engagement.

User and Reservation Management
-Reservation system for table bookings with form validation and secure data storage.
-Role-based user management allows for admin/staff permissions and account management.

Interactive and Responsive UI
-Fully responsive layout across devices (desktop, tablet, mobile). (to be completed)
-Event sliders provide an engaging way to explore upcoming events.
-Smooth hover effects, transitions, and popups improve user experience.

Image Gallery and Slideshow
-Dynamic image gallery for past events.
-Slideshow feature highlights media content for events.

Custom Form Handling
-Contact forms and reservations are validated client-side and server-side.
-Admins can view and manage messages via an inbox system.

Security Best Practices
-Data sanitization and session-based authentication implemented for security.
-Passwords are hashed to protect against vulnerabilities.

Scalable Code Structure
-Modular, reusable back-end code allows for scalability and future feature expansion.



Projektübersicht 
-------------------------------------------------------------------------------------------------
Dynamisches Event-Management-System
-Admins können Events erstellen, bearbeiten, löschen und anzeigen.
-Events werden in kommende und vergangene Events unterteilt und in Echtzeit aktualisiert.
-Benutzer können detaillierte Eventseiten mit Bildern, Beschreibungen und Buchungsoptionen ansehen.

Admin-Dashboard
-Volle Kontrolle über Benutzer, Reservierungen, Menüs und Events.
-CRUD-Funktionalität ermöglicht einfache Verwaltung von Daten.
-Unterstützt Mehrfach-Upload von Bildern für Events zur Verbesserung der Benutzerinteraktion.

Benutzer- und Reservierungsmanagement
-Reservierungssystem für Tischbuchungen mit Formularvalidierung und sicherer Datenspeicherung.
-Rollenbasierte Benutzerverwaltung ermöglicht Admin/Staff-Berechtigungen und Kontoverwaltung.

Interaktives und responsives UI
-Vollständig responsives Layout für alle Geräte (Desktop, Tablet, Mobil). (noch zu erledigen)
-Event-Slider bietet eine ansprechende Möglichkeit, kommende Events zu erkunden.
-Hover-Effekte, Übergänge und Popups verbessern die Benutzererfahrung.

Bildergalerie und Slideshow
-Dynamische Bildergalerie für vergangene Events.
-Slideshow-Funktion hebt Medieninhalte von Events hervor.

Individuelle Formularverarbeitung
-Kontaktformulare und Reservierungen werden clientseitig und serverseitig validiert.
-Admins können Nachrichten über ein Postfachsystem einsehen und verwalten.

Sicherheits-Best-Practices
-Datenbereinigung und sitzungsbasierte Authentifizierung zur Gewährleistung der Sicherheit.
-Passwörter werden gehasht, um Schwachstellen zu vermeiden.

Skalierbare Code-Struktur
-Modularer, wiederverwendbarer Code im Backend ermöglicht Skalierbarkeit und zukünftige Funktionserweiterungen.
